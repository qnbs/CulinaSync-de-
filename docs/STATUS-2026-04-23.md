## Was heute abgeschlossen wurde (2026-04-23)

### Security Fix — CodeQL Alert #7

- `sanitizeWebContentForPrompt` in `geminiService.ts` von fehleranfälligen Regex-Ketten auf `DOMPurify.sanitize()` umgestellt.
- Der schlechte Regex `/<script[\s\S]*?<\/script>/gi` ließ `</script foo="bar">` als validen End-Tag passieren (bekannte Browser-Toleranz → potenzieller XSS/Injection-Pfad).
- `DOMPurify` war bereits als Dependency installiert (v3.3.1) und wird analog zu `recipeImportService.ts` genutzt.
- Downstream-Pipeline (normalizedLines, PROMPT_INJECTION_PATTERN, Truncation) bleibt identisch.

### i18n-Completion Wave 2 + 3 — vollständig

- Alle ~65 verbleibenden hartcodierten deutschen Strings migriert.
- **Komponenten:** `ShoppingListHeader`, `RecipeBookHeader`, `VoiceControlWhisperUI`, `ShoppingListQuickAdd`, `Onboarding` (Tour-Steps), `AiChefPanel` (Suggestions)
- **Services:** `voiceCommands` (Toast-Messages + EN-Keywords), `geminiService` (Error-Messages + language-aware Prompts), `exportService` (MealType-Label-Lookup)
- **Display-Layer:** `DayColumn`, `MealPlanner`, `RecipeDetail` (MealType-Keys korrekt mit `value`-Attribut)
- **Daten:** `foodDatabase.ts` Kategorien per Locale-Lookup; `de/settings.json` Encoding-Fix

### TypeScript 7.0 Beta-Upgrade

- `typescript` von `^5.2.2` auf `^6.0.3` upgradet (stabiler Tooling-Layer für ESLint, Vitest, Storybook).
- `@typescript/native-preview@beta` als neue devDependency hinzugefügt (Go-basierter Compiler, liefert `tsgo`-Binary).
- Build-Script von `tsc && vite build` auf **`tsgo && vite build`** umgestellt.
- Neues `type-check`-Script `tsgo` hinzugefügt — ersetzt `pnpm exec tsc --noEmit` im täglichen Workflow.
- Beide Binaries koexistieren: `tsgo` für Build-Speed, `tsc` (via `typescript@6`) bleibt für Tools die die TypeScript-API direkt konsumieren.

**Gewinn:** bis zu 10× schnellerer Typecheck-Zyklus lokal und in CI, ohne semantische Änderungen oder neue Lint-Ebenen.

### ESLint-Konfigurations-Fix

- Redundanter zweiter Config-Block in `eslint.config.js` entfernt, der `react-hooks/exhaustive-deps` fälschlicherweise auf `'error'` überschrieb.
